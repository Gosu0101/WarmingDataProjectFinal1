  ## Project Summary

  Warming Data Project는 기후변화와 관련된 다양한 공공 데이터를 한 곳에 모아
  **연도별 변화 추이**와 **변수 간 상관관계**를 직관적으로 확인할 수 있도록 만든 데스크톱 분석 프로그램입니다.

  이 프로젝트에서는 단순히 CSV를 출력하는 수준이 아니라,

  - 여러 데이터셋을 공통 기준인 `연도`로 정리하고
  - 시계열 그래프로 추세를 시각화하고
  - 두 변수 간 산점도와 추세선을 그린 뒤
  - 상관계수를 계산하고 해석 문장까지 함께 보여주는

  형태로 분석 흐름을 GUI 안에 통합했습니다.

  ## Why This Project

  기후위기는 더 이상 온도 상승 자체만의 문제가 아니라
  질병, 폭염, 의료비, 감염 매개체 증가 같은 다양한 사회적 문제와 연결됩니다.

  이 프로젝트는 다음 질문에서 출발했습니다.

  - 기온 상승과 폭염일수 증가는 어떤 관계를 보일까?
  - 폭염과 온열질환 환자 수는 실제로 얼마나 같이 움직일까?
  - 온실가스 농도 변화와 기온 지표는 어떤 흐름을 가질까?
  - 모기 개체 수, 호흡기 감염, 계절성 질환 같은 건강 지표는 기후와 어떤 관련이 있을까?

  즉, 여러 데이터를 따로 보는 것이 아니라
  **기후변화와 건강·환경 데이터 간 연결성을 탐색하는 시각화 도구**를 만드는 것이 목표였습니다.

  ## Key Features

  ### 1. 다중 CSV 데이터 로딩
  프로그램 시작 시 여러 공공 데이터 CSV를 자동으로 불러옵니다.

  로드 대상 예시:
  - 온실가스 관측 자료
  - 연도별 기온 요약
  - 폭염일수 요약
  - 연도별 온열질환 합계
  - 감염병 매개 모기 서식 분포
  - 급성 호흡기 감염 통계
  - 계절성 질환 진료비 통계

  ### 2. 시계열 그래프 시각화
  선택한 데이터셋의 연도별 변화를 **라인 차트**로 시각화합니다.

  예:
  - 연도별 평균기온 변화
  - 연도별 폭염일수 변화
  - 연도별 온열질환 환자 수 변화
  - 연도별 온실가스 농도 변화

  ### 3. 상관관계 분석
  두 데이터셋의 특정 컬럼을 선택하면

  - `연도` 기준 데이터 병합
  - 산점도 출력
  - 추세선 표시
  - Pearson 상관계수 계산
  - 상관관계 강도 해석

  까지 한 번에 수행할 수 있습니다.

  ### 4. 결과 해석 자동 출력
  상관계수 수치만 보여주는 것이 아니라
  분석 결과를 다음처럼 사용자 친화적으로 해석합니다.

  - 매우 강한 양의 관계
  - 강한 음의 관계
  - 보통 수준의 관계
  - 거의 관계없음

  ### 5. 데이터 테이블 조회 및 CSV 내보내기
  선택한 데이터셋을 테이블 형태로 확인할 수 있고,
  현재 보고 있는 데이터를 CSV 파일로 다시 저장할 수 있습니다.

  ## Dataset Overview

  프로젝트에 포함된 주요 데이터셋은 다음과 같습니다.

  - `온실가스관측자료.csv`
    - 평균 이산화탄소 농도
    - 평균 아산화질소 농도
  - `연도별_기온_요약.csv`
    - 평균기온
    - 평균최고기온
    - 최고기온 정보
  - `폭염일수_연도별_요약.csv`
    - 연도별 폭염일수
    - 해당 달 평균 최고기온
  - `총괄_연도별_온열질환_합계.csv`
    - 환자 수
    - 추정 사망자 수
  - `감염병매개모기서식분포.csv`
    - 평균 감염병 매개 모기 수
  - `급성호흡기감염증통계.csv`
    - 평균 세균 감염자 수
    - 평균 바이러스 감염자 수
    - 평균 인플루엔자 감염자 수
  - `계절성질환진료비통계.csv`
    - 환자 수
    - 요양급여비용총액

  ## Technical Highlights

  ### 1. 데이터 분석 기능을 GUI에 통합

  이 프로젝트의 핵심은 분석 코드를 Jupyter Notebook에 두는 대신
  일반 사용자도 사용할 수 있는 GUI 프로그램으로 묶었다는 점입니다.

  즉,

  - 데이터 로드
  - 변수 선택
  - 시각화
  - 상관계수 계산
  - 해석 결과 확인

  까지의 흐름을 코드가 아닌 UI 조작으로 수행할 수 있도록 만들었습니다.

  ### 2. 공통 키 기준 데이터 병합

  서로 다른 데이터셋을 직접 비교하기 위해
  각 CSV를 연도 기준으로 병합하는 구조를 적용했습니다.

  이 방식 덕분에

  - 기온 vs 온열질환
  - 폭염 vs 모기 개체 수
  - 온실가스 vs 평균기온

  같은 조합을 유연하게 비교할 수 있습니다.

  ### 3. 분석 결과의 해석 가능성 강화

  단순 수치 출력만으로는 사용자가 분석 결과를 이해하기 어렵기 때문에
  상관계수의 절대값 기준으로 관계 강도를 해석하는 로직을 추가했습니다.

  이를 통해 결과를 더 빠르게 읽을 수 있도록 했습니다.

  ### 4. 모듈 분리 구조

  프로그램은 역할별로 다음처럼 분리되어 있습니다.

  - data_manager.py: 데이터 로딩과 관리
  - graph_maker.py: 그래프 시각화
  - calculator.py: 상관계수 계산과 해석
  - ui_components.py: 메인 윈도우와 UI 연결
  - main.py: 실행 진입점

  규모는 크지 않지만, 기능별 책임을 명확히 나누는 구조를 적용했습니다.
